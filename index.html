FileUpload API

A secure REST API for managing user files — built with Go, Gin, and GORM.

Live API GitHub Repo

📦 About the API

The FileUpload API allows users to upload, manage, and restore files with role-based access control. It supports soft deletes, idempotent uploads, and administrative recovery.

  • Upload multiple files per request (UUID-tied)
  • Idempotent uploads — safe against network failures
  • Soft delete & recovery (within 30 days)
  • Admin & Super Admin role hierarchy

🔐 Authentication

All secure endpoints require JWT authentication via the header:

Authorization: Bearer <token>

🧾 Endpoints Overview

MethodEndpointDescription
POST/api/registerRegister a new user
POST/api/loginLogin and receive JWT token
POST/api/files/uploadUpload one or more files
GET/api/filesGet all files for authenticated user
GET/api/files/:idFetch a single file
DELETE/api/files/:idSoft delete a file
PATCH/api/files/restore/:idAdmin restores a deleted file
GET/api/admin/usersFetch all users (admin only)
PATCH/api/admin/users/restore/:emailRestore soft-deleted user

🧑‍💼 Admin & Super Admin

Super Admin is created via CLI seeding (using --seed-admin) with credentials from environment variables.

  • Fetch and manage users
  • Promote users to admin
  • Set upload limits per user or globally
  • View and restore deleted files/users

🚀 Future Roadmap
